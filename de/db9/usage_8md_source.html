7 ## Initilizing the Library
8 
9 You can initilize the library in two ways: with hardware SPI or with software SPI.
10 
11 1. Hardware SPI is at least 10 times faste but you can only use specific pins for some signals (look [here](index.html) for which pins).
12 2. Software SPI is slower (which only mattesrs if you need really tight timings) but you can freely choose which (free) pins you want to use.

14 ### Small introduction to SPI
15 
16 The Serial Peripheral Interface (SPI) is a synchronous serial communication interface specification used for short-distance communication, primarily in embedded systems.
17 In this case your microcontroller is the master which controls the slave (at least one 7-Segment display or led matrix).
18 
19 The master has three significant pins: SCLK(Serial Clock), MOSI(Master Out Slave In), SS(Slave Select).
20 The slave also has tree needed pins (and two for power(GND and VCC)): CLK(Clock), DIN(Data In), CS(Ship Select).
21 SCLK has to be connected to CLK and MOSI to DIN.
22 These signals can be shared by multiple SPI slaves even if they have nothing to do with each other.
23 SS hat to be connected to CS and each Slave has its own SS pin(which can be any free pin).

25 ### Software SPI
26 
27 The constructor for the LedController needs at least 3 arguments but accepts five.
28 
29 1. unsigned int dataPin: The Pin which should be used for the MOSI signal.
30 2. unsigned int clkPin: The pin which should be used for the SCLK signal.
31 3. unsigned int csPin: The pin which should be used for the SS signal.
32 4. (optional default 4) unsigned int numSegments: if your amount of segments differs from 4, specify the number.
33 5. (optional default false) bool useHardwareSpi: set to true if you want to use hardware SPI with the specified ports.
34 
35 Simply declare a global object with these parameters and you are ready to go.
